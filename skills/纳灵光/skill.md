# 纳灵光 — 日常思考结构化入库

## 概述

纳灵光处理一类特殊的"知识来源"——用户的日常思考和直觉。它们不同于原始资料（没有来源文件）、不同于阅读批注（没有依附的文本），而是用户脑子里自发产生的、模糊但有价值的想法。本 skill 的任务是：**提炼、结构化、融入既有知识体系**。

核心原则：**思考本身不单独成类，而是作为养分注入已有知识页面**。

## 知识库路径

```
Vault: E:\obsidian\potato
结构规范: E:\obsidian\potato\CLAUDE.md
观点索引: E:\obsidian\potato\10.知识库\观点索引.md
操作日志: E:\obsidian\potato\log.md
模板目录: E:\obsidian\potato\模板\
```

## 三种运行模式

### 模式判定

收到用户输入后，首先判断用户的表达状态：

```
用户输入
    ↓
┌─ 表述模糊、不完整、"说不清" → 追问模式
│
├─ 表述完整、有推理链、有结论 → 快速入库
│
└─ "直觉觉得有意思""值得聊聊""不知道怎么看" → 探索模式
```

三个模式之间不设硬边界。当追问中发现值得展开的角度，可自然滑入探索；当探索中想法逐渐成形，可自然收束为写入计划。

---

### 模式一：追问模式

**适用**：用户大致知道自己在想什么，但表述不够清晰、完整。

**LLM 姿态**：提炼者——帮用户挖出已有但未言明的想法。

**行为指引**：
1. 识别用户表述中的**核心概念**和**逻辑断点**
2. 提出聚焦性问题帮助用户深化（不是发散，是聚焦）
3. 每次追问后，用一句话复述你对用户想法的理解，确认方向正确
4. 当能够提取出"核心洞见 + 关键词 + 领域关联"时，判断成熟，转入撰写写入计划
5. 如果追问 2-3 轮后用户仍难以确定方向，不要强行追问——给出当前最好的理解，用写入计划呈现，标注不确定之处供用户修正

**追问示例**：
```
用户：我觉得摄像头这个东西在权力关系上很有意思
LLM：你是关注摄像头作为"被观看"的规训体验，还是它作为国家权力符号
     在不同文化中的感知差异？前者更偏福柯/社会学，后者涉及跨文化传播和政治学。
```

---

### 模式二：探索模式

**适用**：用户只有一个模糊直觉——感觉某个话题有探索价值，但完全不清楚自己最终的结论会是什么。

**LLM 姿态**：辩论者——提供外部视角、挑战预设、补充盲区。你的任务是激发用户思考，不是替用户思考。

**行为指引**：
1. 围绕话题，从**不同角度**（学科视角、立场视角、历史视角）提出有价值的见解或问题
2. 可以挑战用户的预设——"你有没有想过，这个想法的前提本身可能有问题？"
3. 记录对话中的关键论点（用户的和你的都记录），在探索结束时用于总结
4. **控制节奏**：每次只抛出 1-2 个角度，等用户回应后再推进，不要连续输出长篇分析
5. 当用户说"差不多了""可以总结了"或讨论自然收束时，切换为整理角色——总结对话中的核心洞见，产出写入计划

**重要的来源标注**：探索模式中 LLM 输出的见解（非用户原创），在写入计划中应明确标注"经由讨论激发"或"LLM 提出"，供用户决定保留、修改或删除。入库时用户原创观点与 LLM 激发观点不混同。

---

### 模式三：快速入库

**适用**：用户表述完整——有推理链、有结论、甚至可能已暗示了领域归属。

**行为指引**：
1. 直接进入分析：提取核心洞见 → 关键词 → 领域映射
2. 产出写入计划，跳过追问环节
3. 如果分析过程中发现关键模糊之处，回退到追问模式

---

## 分析框架

无论哪种模式，在产出写入计划前，必须完成以下分析：

### 1. 提取核心洞见

从用户表述中提炼 1-N 个核心洞见。洞见应当：
- 是一句完整的判断，而非一个词组
- 具有可迁移性——不是"摄像头在中国怎么了"，而是"权力符号在不同文化中的感知差异"
- **可以有多个**：发散的思考可能同时涉及几个不同的洞察方向

### 2. 提取关键词

5-10 个关键词，用于定位领域和标签。

### 3. 映射领域

基于 vault 的 CLAUDE.md 第三章（六大知识领域），判断该思考涉及哪些领域。跨领域思考可以同时映射到多个领域。

**映射原则**：
- 选**主要面向**的领域作为物理存放位置的参考
- 通过 tags 标注所有涉及的领域
- 跨领域本身就是价值——不要强行归入单一领域

### 4. 检索既有知识

**在产出写入计划前，必须读取相关已有页面**。检索采用三层递进策略，以控制上下文消耗为核心约束——不泛读、不扫库、不凭记忆猜测。

#### 第一层：粗筛（MOC + 索引页）

先读取以下页面，获取知识库的结构全貌：
- 相关领域的 MOC 页面（路径参见 vault 的 CLAUDE.md 第三章）
- `观点索引.md` —— 检查是否有相似观点已被记录

此层只读索引/导航页，不读具体知识页。目标：将候选页面范围从可能的 20+ 个压缩到 **2-5 个**。

#### 第二层：精读 + 重复检测

从粗筛结果中选出 **2-3 个最相关的页面**，读取完整内容。精读时同时执行两项任务：

**任务一：定位写入点**
- 了解目标页面的内容、格式和链接结构
- 判断新思考可补充哪个页面的哪个段落、是否值得创建新页面（慎重）

**任务二：重复检测**
- 将新思考的核心洞见与已有页面内容做逐条比对
- 判定重复级别并据此决定处理方式：

| 重复级别 | 判定标准 | 处理方式 |
|----------|----------|----------|
| 高度重叠 | 核心判断已被已有页面覆盖，新思考无实质增量 | 告知用户"此思考已在 [[页面名]] 中覆盖"，建议改为补充链接或放弃入库。**不追加重复内容**。 |
| 部分重叠 | 同主题但不同角度/层次，或提供了新的例证 | 标注"与 [[页面名]] 存在交集"，说明差异点，计划以补充角度写入。 |
| 无重叠 | 已有知识中未覆盖此洞见 | 正常推进写入计划。 |

重复检测的底线：**宁可告知用户"已覆盖"让用户决定是否仍要补充，也不静默写入重复内容**。

#### 第三层：扩展检索（按需）

仅当精读页面后发现有跨领域连接线索、且连接指向的页面尚未被读取时，才额外读取。此层不超过 1-2 个额外页面。无明确连接线索时跳过此层。

## 写入计划

讨论成熟后，产出结构化写入计划供用户确认。

### 写入计划格式

```markdown
## 写入计划

### 核心洞见
1. **[洞见一]** —— 一句话概括 + 来源标注
   - 来源：用户原创 / 讨论激发 / LLM 提出
2. **[洞见二]** —— ...

### 关键词
`关键词1` `关键词2` `关键词3` ...

### 领域归属
🏛️ 人文社科/新闻传播 | 🏛️ 人文社科/社会学 | ...

### 写入操作

**补充已有页面**（优先）：
1. [补充] `已有页面名.md` —— 追加内容描述 + 计划追加的段落概要
2. [补充] `已有页面名.md` —— ...

**新建页面**（仅当足够抽象可迁移）：
1. [新建] `新页面名.md` —— 所属领域 + 建页理由 + 内容概要

**索引更新**：
- [更新] `观点索引.md` —— 记录思条目
- [更新] `log.md` —— 追加操作日志

### 跨领域链接建议
- [[页面X#章节锚点]] ← → [[页面Y]] —— 连接理由 + 具体链接位置
```

### 写入优先级原则

1. **补充已有页面优先**：大多数日常思考应该追加到已有页面的合适位置，而非创建独立页
2. **创页从紧**：只有抽象、可迁移的概念/洞见才值得独立建页。具体案例、事件、举例不应创建独立页
3. **匹配既有格式**：追加内容必须匹配目标页面的既有格式和风格。写之前先读目标页面

### 用户确认

写入计划产出后，等待用户确认。用户可以：
- 直接批准执行
- 修改某些写入操作
- 删除某些洞见
- 要求调整领域归属

---

## 执行写入

用户确认后，按写入计划执行：

### 补充已有页面

1. 读取目标页面完整内容
2. 在合适位置追加新内容，保持与原有格式一致
3. 确保双向链接：新追加的内容中建立到相关页面的 wikilink；如果涉及的概念页需要回链，也一并更新
4. 更新页面的 `updated` 日期

### 创建新页面

1. 参考 `模板/` 目录下的对应模板（概念页、对比页等）
2. 完整填写 frontmatter（title, domain, area, type, tags, created, updated, aliases, status）
3. 内容结构参照同领域已有页面的格式
4. 建立双向链接

### 更新观点索引

在 `观点索引.md` 中追加新条目，格式与既有条目一致：
```markdown
### 关于XXX

> "用户的原始洞见或精炼后的表述"

— [[来源或相关页面]]，简短说明背景和连接。
```

### 更新操作日志

在 `log.md` 顶部追加，使用既有格式：
```markdown
## [YYYY-MM-DD] 纳灵光 | 标题

- **操作**：纳灵光
- **领域**：涉及的领域
- **涉及页面**：
  - [[页面路径|页面名]] —— 操作描述
- **摘要**：简短总结本次入库的思考内容
```

### 更新 MOC

如果新建了页面，将新页面加入对应 MOC 的手工分组。如果只是补充已有页面且未改变页面列表，MOC 无需更新。

---

## 注意事项

- **不创建单独的"日常思考"类别或文件夹**。思考入库后完全融入既有知识体系，不保留"这是思考"的标记（除了观点索引中的记录）
- **观点索引是唯一的"来源"记录**——它标记了哪些观点来自用户原创思考
- **写入内容与 LLM 参与部分要可区分**：探索模式中 LLM 贡献的见解，在写入计划中标注来源，最终是否写入及如何措辞由用户决定
- **先读后写**：分析阶段必须读取相关已有页面，不凭记忆或猜测已有内容
- **格式从旧**：新内容严格参照目标页面的既有格式，不引入新的格式风格
- **跨领域连接是核心价值**：日常思考的价值往往在于"连接了看似不相关的东西"——写入计划中跨领域链接建议不应为空
