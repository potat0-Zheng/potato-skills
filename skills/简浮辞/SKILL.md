# 简浮辞 — 中文语义压缩

剥离浮词，沉淀实义。启发自 caveman（英文压缩），针对中文语法全新设计。

## 与 caveman 的本质区别

| 维度 | caveman（英文） | 简浮辞（中文） |
|------|----------------|---------------|
| 分词基础 | 天然空格分界 | 依赖 jieba 分词 |
| 剥离目标 | 冠词/助动词/冗余介词 | 助词（的得地）/语气词/冗余副词/形式动词 |
| 保留核心 | 名词+主动词 | 名词+核心动词+否定/情态/量词 |
| 语法结构 | 主谓宾，助动词标记 | 话题-述题，虚词标记 |

caveman 的规则一个都不能直接搬用。中文没有 a/an/the、没有 is/are/was/were、没有动词变位。

## 压缩原理

基于 jieba 词性标注（PKU 词性集），对每个词做"保留/删除"二元决策：

### 一律保留
名词（n）、人名（nr）、地名（ns）、机构名（nt）、专名（nz）、动词（v）、数词（m）、时间词（t）、空间方位词（f）

### 情境决定
- **副词**：保留否定（不、没）和量化副词（至少、约），去掉冗余强调（很、非常、极其）
- **介词**：保留空间/方向（在、从、到），去掉语法工具（被、把、将）
- **连词**：保留逻辑关系（因为、如果），去掉顺承关系（然后、并且）
- **代词**：保留疑问/指示（什么、这），人称代词视预设等级决定
- **形容词**：单字修饰去掉，双字及以上保留

### 一律删除
语气词（吧、吗、呢、啊）、拟声词、叹词、冗余"的/了/着/过"、形式动词（进行、加以）、冗余量词头（一个、这种）

## 安装

```bash
pip install jieba
# 可选：语义校验
pip install sentence-transformers
```

## 使用

```bash
# 中度压缩（默认），输出到 stdout
python scripts/compress.py input.txt

# 轻度压缩（保留更多可读性）
python scripts/compress.py input.txt -p light

# 重度压缩（最大压缩比）
python scripts/compress.py input.txt -p heavy

# 输出到文件
python scripts/compress.py input.txt -o compressed.txt

# 启用语义相似度校验
python scripts/compress.py input.txt -p heavy --check

# 管道输入
echo "这个系统被设计用于高效地处理大规模数据。" | python scripts/compress.py -

# 仅看统计，不输出文本
python scripts/compress.py input.txt -s

# 逐词调试模式
python scripts/compress.py input.txt -v
```

## 三级压缩预设

| 预设 | 压缩比目标 | 剥离内容 | 可读性 |
|------|-----------|---------|--------|
| `light` | ~85% | 仅冗余副词和语气词 | 高，接近原文 |
| `medium` | ~65% | 上述 + 虚词助词 + 顺承连词 | 中，基本可读 |
| `heavy` | ~45% | 上述 + 代词 + 单字修饰词 | 低，语义骨架 |

## 架构

```
scripts/
  compress.py     → 核心压缩引擎（~300 行，7 层流水线）
  rules.json      → 剥离规则表（POS + 例外词 + 预设配置）
  requirements.txt → 依赖声明
```

### 7 层流水线

| 层 | 功能 | 实现方式 |
|----|------|---------|
| 1 | 文本分段分句 | 正则按句末标点 + 换行分割 |
| 2 | 分词 + 词性标注 | jieba.posseg (PKU 词性集) |
| 3 | 实体识别 | 基于词性 nr/ns/nt/nz（不依赖 HanLP） |
| 4 | 删除权重计算 | POS 规则 + 预设等级 + 显式例外词表 |
| 5 | 重建拼接 | 规则拼接，处理标点粘连和拉丁字符间距 |
| 6 | 语义校验 | 可选，sentence-transformers 余弦相似度 |
| 7 | 连贯性修复 | 跨句去重、连续标点清理、空行归一 |

## 适用场景

- **Prompt 精简**：把冗长的中文 system prompt 或 few-shot 示例压进更小的 token 窗口
- **长文档脱水**：提取核心语义骨架，快速概览内容
- **上下文窗口优化**：多轮对话中压缩历史消息以节省 token
- **嵌入式场景的输入预处理**：在送入 embedding 模型前剥离噪音词

## 不适用场景

- 需要精确措辞的正式文档（合同、公告）
- 文学创作（会丢失语气和修辞效果）
- 代码或结构化数据
